# MarkItDown Cloud Web App

واجهة عربية بسيطة فوق محرك MarkItDown الأصلي، مصممة للنشر السحابي دون تثبيت الأداة على جهاز المستخدم.

## ما الذي توفره؟

- رفع ملف واحد أو عدة ملفات.
- تحويل كل ملف إلى Markdown باستخدام `convert_local()` فقط.
- معاينة أول نتيجة ناجحة داخل المتصفح.
- تنزيل ملف `.md` مباشرة عند تحويل ملف واحد.
- تنزيل ZIP يتضمن النتائج وتقرير الأخطاء عند التحويل الدفعي.
- استمرار العملية عند فشل ملف واحد بدل إلغاء الدفعة كاملة.
- تنظيف تلقائي للنتائج المؤقتة القديمة.
- حدود قابلة للتعديل لعدد الملفات وأحجامها.

> الهدف هو استخراج المحتوى والبنية المهمة للذكاء الاصطناعي وقواعد المعرفة، وليس إعادة إنتاج التصميم البصري الأصلي بنسبة 100%.

## التشغيل باستخدام Docker

```bash
docker build -f Dockerfile.web -t markitdown-web .
docker run --rm -p 7860:7860 markitdown-web
```

ثم افتح `http://localhost:7860`.

## النشر السحابي من GitHub

يمكن نشر المستودع مباشرةً على أي منصة تدعم Docker، مثل Render أو Railway أو Azure Container Apps أو Hugging Face Docker Spaces، مع اختيار:

- Dockerfile: `Dockerfile.web`
- Port: `7860`، أو اترك المنصة تمرر متغير `PORT` تلقائيًا.
- Health path: `/`

## الإعدادات المرنة

| المتغير | الافتراضي | الغرض |
|---|---:|---|
| `MAX_FILES` | `20` | أكبر عدد ملفات في العملية |
| `MAX_FILE_MB` | `50` | أكبر حجم للملف الواحد |
| `MAX_TOTAL_MB` | `200` | أكبر حجم إجمالي للدفعة |
| `OUTPUT_RETENTION_HOURS` | `12` | مدة الاحتفاظ المؤقت بالنتائج |
| `ALLOW_ARCHIVES` | `false` | السماح بملفات ZIP |
| `PORT` | `7860` | منفذ خادم الويب |

## اعتبارات الأمان والخصوصية

- الواجهة لا تقبل روابط URL؛ التحويل محصور في الملفات المرفوعة محليًا إلى الحاوية.
- ملفات ZIP معطلة افتراضيًا لتقليل مخاطر الأرشيفات المضغوطة الضارة.
- أسماء الملفات تنظف قبل استخدامها في المخرجات.
- حجم وعدد الملفات محدودان وقابلان للتعديل.
- النتائج تحفظ في مجلد مؤقت وتحذف بعد انتهاء مدة الاحتفاظ.
- عند استخدام خدمة استضافة خارجية، تطبق سياسات الخصوصية والاحتفاظ الخاصة بمزود الاستضافة كذلك.

## تحسينات مستقبلية مقترحة

- حماية بكلمة مرور أو تسجيل دخول.
